# C1 Spanish Grammar Trainer — Product & Technical Specification

## 1. Product Overview

A single-user web application for practicing C1-level Spanish grammar. The app generates quiz rounds of 10 questions using the Claude API (Sonnet), grades answers with AI, tracks all results in PostgreSQL, and provides a full analytics dashboard showing accuracy by topic, improvement trends, weak areas, and session history.

### 2.1 Dashboard (Landing Page)
- Summary stats: total quizzes taken, overall accuracy %, current streak (consecutive days with at least 1 quiz)
- Quick chart: accuracy trend over last 30 days (line chart)
- Weakest 3 topics with accuracy %
- **"Start Quiz"** button (prominent)
- Nav links: History, Statistics

### 2.2 Topic Picker
After clicking "Start Quiz", the user sees:
- **"Let AI pick my weak areas"** (default) — the system selects topics where accuracy is lowest or least practiced
- **Manual topic selection** — checkboxes for specific grammar topics (see §3)
- **"Start"** button to begin the round

### 2.3 Quiz Round (10 Questions)
- Questions are displayed one at a time with a progress indicator (e.g., "3/10")
- Three question formats mixed within a round (see §4)
- User answers via dropdown selection or text input depending on format
- **No instant feedback** — answers are submitted at the end of the round
- "Next" button to advance; "Previous" button to go back and change answers
- "Submit" button on question 10

### 2.4 Results & Explanations
After submission:
- Show score: X/10
- List all 10 questions with:
  - ✅ or ❌ indicator
  - The user's answer vs. the correct answer
  - For **wrong answers**: explanation shown by default (generated by AI during question creation)
  - For **correct answers**: "Show explanation" toggle (collapsed by default)
- **"New Quiz"** button to start another round
- **"Back to Dashboard"** button

## 3. Grammar Topics

All topics are C1 level. Each question is tagged with exactly one topic.

| ID | Topic | Description |
|----|-------|-------------|
| `subjuntivo_presente` | Subjuntivo Presente | Triggers, formation, irregular forms |
| `subjuntivo_imperfecto` | Subjuntivo Imperfecto | -ra/-se forms, hypothetical clauses |
| `subjuntivo_pluscuam` | Subjuntivo Pluscuamperfecto | Counterfactuals, wish expressions |
| `subjuntivo_triggers` | Subjuntivo Triggers | WEIRDO, conjunctions (antes de que, para que, etc.) |
| `ser_estar` | Ser vs Estar | Advanced: passive, resultant state, meaning changes |
| `ser_estar_haber` | Haber Existential | Hay/había/habrá, common errors |
| `por_para` | Por vs Para | Advanced distinctions, fixed expressions |
| `verbal_regime` | Verbal Regime | Preposition governed by verb (depender de, insistir en, etc.) |
| `conditionals` | Conditionals | Mixed conditionals, si + pluperfect subjunctive |
| `relative_clauses` | Relative Clauses | Indicative vs subjunctive in relatives, el que/quien/cuyo |
| `reported_speech` | Reported Speech | Tense backshift, reporting verbs |
| `connectors` | Discourse Connectors | No obstante, sin embargo, a pesar de que, etc. |
| `perifrasis` | Periphrastic Verbs | Ir a + inf, llevar + gerund, acabar de + inf, etc. |
| `passive_voice` | Passive Constructions | Ser + past participle, passive se, impersonal se |
| `mixed` | Mixed / General | Cross-topic questions |

## 4. Question Formats

### 4.1 Multiple Choice (Dropdown)
A sentence with a blank. 4 options provided in a dropdown. Exactly one is correct.

```json
{
  "format": "multiple_choice",
  "topic": "subjuntivo_presente",
  "stem": "No creo que ellos ___ la verdad.",
  "options": ["saben", "sepan", "sabrán", "supieran"],
  "correct_answer": "sepan",
  "explanation": "After 'no creo que' (negated belief), the subjunctive is required. Present subjunctive of 'saber' → 'sepan'."
}
```

### 4.2 Fill-in-the-Blank (Text Input)
A sentence with a blank. The user types the answer. Graded by AI.

```json
{
  "format": "fill_in_blank",
  "topic": "por_para",
  "stem": "Salimos temprano ___ no llegar tarde.",
  "correct_answer": "para",
  "acceptable_answers": ["para"],
  "explanation": "'Para' expresses purpose (in order to). 'Por' would indicate cause/reason, not purpose here."
}
```

### 4.3 Cloze Passage
A paragraph (3–6 sentences) with 2–4 blanks. Each blank is either dropdown-based or free-form.

```json
{
  "format": "cloze",
  "topic": "conditionals",
  "passage": "Si yo ___(1) más tiempo, ___(2) contigo al cine. Pero ayer ya te ___(3) que no podía.",
  "blanks": [
    { "id": 1, "type": "free", "correct_answer": "tuviera", "explanation": "..." },
    { "id": 2, "type": "dropdown", "options": ["iría", "voy", "fui", "iba"], "correct_answer": "iría", "explanation": "..." },
    { "id": 3, "type": "free", "correct_answer": "dije", "explanation": "..." }
  ]
}
```

### 4.4 Distribution per Round
A round of 10 questions should contain approximately:
- 4 multiple choice
- 3 fill-in-the-blank
- 3 cloze blanks across 1–2 cloze passages (a passage with 3 blanks counts as 3 questions)

Total question count visible to user = 10 (cloze blanks count individually toward the 10).

## 5. AI Integration (Claude API)

### 5.1 Question Generation
- **Model**: `claude-sonnet-4-6` (via Anthropic SDK)
- **When**: On every "Start Quiz" — generate all 10 questions in a single API call
- **Prompt strategy**: System prompt establishes the role as a C1 Spanish grammar examiner. The user message specifies:
  - Which topics to cover (from topic picker or adaptive selection)
  - The format distribution (§4.4)
  - Required JSON output schema
  - Instruction to vary difficulty and avoid repetition of recent stems (pass last 20 stems from DB as context)
- **Response format**: Structured JSON matching the schemas in §4
- **Validation**: Server-side validation of the returned JSON before presenting to user. If validation fails, retry once.

### 5.2 Answer Grading (Fill-in-the-Blank & Free Cloze)
- **Model**: `claude-sonnet-4-6`
- **When**: On quiz submission, for each free-text answer
- **Prompt**: Given the stem, the user's answer, and the expected correct answer, determine if the user's answer is correct. Account for:
  - Accent variations (e.g., "el" vs "él" — but flag if accent changes meaning)
  - Alternative correct forms (e.g., "tuviera" and "tuviese")
  - Minor typos (1 character off) — mark correct but note the typo
- **Response**: `{ "correct": true/false, "note": "optional note about typos or alternatives" }`

### 5.3 Adaptive Topic Selection
When user selects "Let AI pick my weak areas":
- Query DB for accuracy by topic (last 30 days)
- Select topics with lowest accuracy or fewest attempts
- Weight: 60% weak topics, 20% medium topics, 20% random (to maintain breadth)
- No AI call needed — this is pure DB query + algorithm

### 5.4 API Key Configuration
- Stored as environment variable: `ANTHROPIC_API_KEY`
- Loaded server-side only, never exposed to client

## 6. Database Schema (PostgreSQL)

Connection: `DATABASE_URL=postgresql://username:password@localhost:5432/c1`

### 6.1 Tables

```sql
-- Quiz sessions
CREATE TABLE sessions (
    id              SERIAL PRIMARY KEY,
    started_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    completed_at    TIMESTAMPTZ,
    topic_mode      VARCHAR(20) NOT NULL,  -- 'adaptive' or 'manual'
    selected_topics TEXT[],                 -- topics selected (NULL if adaptive)
    score           INT,                    -- correct count (set on completion)
    total           INT NOT NULL DEFAULT 10
);

-- Individual questions within a session
CREATE TABLE questions (
    id              SERIAL PRIMARY KEY,
    session_id      INT NOT NULL REFERENCES sessions(id) ON DELETE CASCADE,
    position        INT NOT NULL,           -- 1-10 within the session
    format          VARCHAR(20) NOT NULL,   -- 'multiple_choice', 'fill_in_blank', 'cloze'
    topic           VARCHAR(50) NOT NULL,   -- topic ID from §3
    stem            TEXT NOT NULL,           -- question text / passage
    options         JSONB,                  -- dropdown options (NULL for fill_in_blank)
    correct_answer  TEXT NOT NULL,           -- expected answer (for cloze: JSON array)
    explanation     TEXT NOT NULL,
    question_data   JSONB NOT NULL          -- full AI-generated question JSON
);

-- User answers
CREATE TABLE answers (
    id              SERIAL PRIMARY KEY,
    question_id     INT NOT NULL REFERENCES questions(id) ON DELETE CASCADE,
    user_answer     TEXT NOT NULL,
    is_correct      BOOLEAN NOT NULL,
    ai_note         TEXT,                   -- grading note (typos, alternatives)
    answered_at     TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Indexes
CREATE INDEX idx_sessions_started_at ON sessions(started_at);
CREATE INDEX idx_questions_session_id ON questions(session_id);
CREATE INDEX idx_questions_topic ON questions(topic);
CREATE INDEX idx_answers_question_id ON answers(question_id);
CREATE INDEX idx_answers_is_correct ON answers(is_correct);
```

### 6.2 Cloze Storage
For cloze passages, each blank is stored as a separate row in `questions` with the same `stem` (the full passage). The `position` reflects the blank's order in the 10-question round. `correct_answer` holds the single blank's answer. `question_data` holds the full cloze JSON including all blanks (duplicated across rows for simplicity).

## 7. Statistics & Analytics

### 7.1 Dashboard Summary (Landing Page)
| Metric | Query Logic |
|--------|-------------|
| Total quizzes | `COUNT(*) FROM sessions WHERE completed_at IS NOT NULL` |
| Overall accuracy | `SUM(is_correct) / COUNT(*) FROM answers` |
| Streak (days) | Count consecutive days with ≥1 completed session, going back from today |
| 30-day trend | Daily accuracy for the last 30 days |
| Weakest 3 topics | Bottom 3 topics by accuracy % (min 5 answers) |

### 7.2 Statistics Page
- **Accuracy by topic**: Bar chart — one bar per topic, showing % correct. Include attempt count.
- **Trend over time**: Line chart — weekly rolling average accuracy, overall and per-topic filter.
- **Format breakdown**: Accuracy by question format (MC vs fill-in vs cloze).
- **Session history table**: Date, score, topics, duration. Clickable to review individual sessions.
- **Totals**: Total questions answered, total sessions, total time (derived from session start/complete timestamps).

### 7.3 History Page
- Paginated list of past sessions (newest first)
- Each row: date, score (X/10), topics, link to review
- **Session review**: Same layout as Results (§2.4) — see all questions, answers, and explanations

## 12. AI Prompt Templates

### 12.1 Question Generation System Prompt

```
You are a C1-level Spanish grammar examiner creating practice questions for the DELE C1 exam.

Rules:
- All sentences must be natural, authentic Spanish — the kind found in newspapers, essays, or formal conversation.
- Questions must test genuine C1-level grammar knowledge, not vocabulary.
- Each question must have exactly one unambiguous correct answer.
- Explanations must be concise (2-3 sentences), in English, and reference the specific grammar rule.
- For cloze passages, use coherent mini-texts (not isolated sentences).
- Vary difficulty within the round.
- Do not reuse stems from the "recent stems" list provided.
```

### 12.2 Question Generation User Prompt Template

```
Generate a quiz round of 10 questions for C1 Spanish grammar practice.

Topics to cover: {topics_list}
Format distribution: 4 multiple choice, 3 fill-in-the-blank, 1-2 cloze passages (totaling 3 blanks)

Recent stems to avoid (do not reuse these):
{last_20_stems}

Respond with a JSON object matching this schema:
{json_schema}
```

### 12.3 Grading Prompt Template

```
You are grading a C1 Spanish grammar answer.

Question: {stem}
Expected answer: {correct_answer}
User's answer: {user_answer}

Evaluate if the user's answer is correct. Consider:
- Accent marks: flag if missing but meaning doesn't change; mark wrong if accent changes meaning
- Alternative conjugation forms (e.g., -ra/-se for imperfect subjunctive) are both correct
- Typos within 1 character: mark correct but note the typo

Respond with JSON: { "correct": boolean, "note": "optional string" }
```

## 13. Edge Cases & Error Handling

| Scenario | Handling |
|----------|----------|
| AI returns invalid JSON | Retry once. If still invalid, show error and offer to retry quiz generation. |
| AI timeout | 30-second timeout. Show error message, offer retry. |
| Empty DB (first use) | Dashboard shows welcome message + "Start your first quiz" CTA. Stats show "No data yet." |
| User navigates away mid-quiz | Session stays in DB with `completed_at = NULL`. Incomplete sessions excluded from stats. |
| Accent-insensitive grading | Handled by AI grading (§5.2), not by string comparison. |
| Cloze passage has fewer blanks than expected | Validate AI output; if blank count doesn't match, adjust question count or regenerate. |
| API key missing/invalid | Show setup instructions on first load if API calls fail with auth error. |

## 14. UI/UX Notes

- **Design**: Clean, minimal. Dark/light mode toggle. Spanish-themed accent colors.
- **Typography**: Clear, readable. Monospace for fill-in-the-blank inputs.
- **Mobile**: Responsive. Quiz must be comfortable on mobile.
- **Loading states**: Skeleton loaders during AI generation (can take 3-8 seconds). Show "Generating your quiz..." message.
